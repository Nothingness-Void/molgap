## Decision Table

| Idea | Decision | Why |
|---|---|---|
| Full replacement300K standard hybrid | Adopt now | 30K common eval gives a weak-positive hard-slice signal and the run is feasible. |
| Warm-start replacement300K encoders from Phase 7 weights | Adopt now as the first full run | Literature supports DFT/GW transfer learning; only 38,620/300,000 rows changed, so warm-start should preserve v1 knowledge and reduce training time/forgetting. |
| From-scratch replacement300K encoders | Keep as audit run | Cleaner isolation of data coverage, but costs more time. Run only if warm-start result is ambiguous. |
| MoE | Reject for now | Frozen-head gain <=0.0006 eV; end-to-end 30K MoE worsened Gap MAE by about 0.011 eV; no speed advantage. |
| IDLe / masked multi-fidelity heads | Defer | Technically aligned but conflicts with the current need to keep a strong B3LYP model; also needs end-to-end multi-task training and careful loss weighting on scarce GW labels. |
| SchNet denoising pretraining | Backlog after v2 | Zaidi-style denoising is the best pretraining candidate, but it should not interrupt the already-justified replacement300K run. |
| MoleVers-style DFT auxiliary pretraining | Conceptually already covered | MolGap already has abundant B3LYP HOMO/LUMO/Gap labels; the practical version is better warm-start/finetune, not adding a new auxiliary-label stage now. |
| AutoGNNUQ-style uncertainty decomposition | Keep for delivery layer | Phase 10 already has ensemble + k-NN UQ; do not add NAS. Revalidate after v2. |
| GraphLoRA / ELoRA | Keep for GW refinement | LoRA pilots already show value. Re-run on the chosen v2 base, not before. |
| Green's-function direct GW surrogate | Reject for production | Requires mean-field features / per-molecule quantum calculation, breaking the zero-DFT high-throughput value proposition. Good for defense only. |
| QM9GWBSE / QM9-GW direct merge | Reject for current labels | Large but small-molecule/QM9 and fidelity mismatched; can be benchmark/reference, not mixed into current GW target. |
| OMol25 / Force2Geo geometry path | Defer | Geometry diagnostics already show ETKDG vs PBE geometry only changes about 0.01-0.02 eV. Not the current bottleneck. |

## Immediate Change Made

Added `--init-from` to `scripts/phase8/train_encoder.py`.

This allows tomorrow's replacement300K encoder training to start from the Phase 7
300K checkpoints without forking model code:

```powershell
.venv\Scripts\python.exe scripts\phase8\train_encoder.py `
  --kind gps `
  --graphs results\phase8\pyg_2d_graphs_bond_replacement_300k.pt `
  --model-out models\phase8_gps_replacement_300k_warm.pt `
  --metrics-out results\phase8\gps_replacement_300k_warm_metrics.json `
  --embeddings-out results\phase8\gps_replacement_300k_warm_embeddings.pt `
  --init-from models\gps_2d_300k.pt

.venv\Scripts\python.exe scripts\phase8\train_encoder.py `
  --kind schnet `
  --graphs results\phase8\pyg_3d_graphs_etkdg_replacement_300k.pt `
  --model-out models\phase8_schnet_replacement_300k_warm.pt `
  --metrics-out results\phase8\schnet_replacement_300k_warm_metrics.json `
  --embeddings-out results\phase8\schnet_replacement_300k_warm_embeddings.pt `
  --init-from models\gnn_schnet_3d_300k.pt
```

Then train the standard single `FusionHead` on the warm-start embeddings. Keep
the Phase 7 300K model as the control and evaluate on the same OOD/hard sets.

## Recommended Next Experiment Order

1. Build full replacement300K graph caches if not already present.
2. Train warm-start GPS and SchNet on replacement300K using `--init-from`.
3. Train standard single `FusionHead` on the warm-start embeddings.
4. Evaluate against Phase 7 v1 on:
   - Phase 7 OOD-1000;
   - P8 targeted hard slice;
   - internal replacement test split.
5. If warm-start improves hard slice without OOD regression, accept as v2
   candidate.
6. If warm-start is neutral or worse, run a from-scratch replacement300K audit
   before declaring replacement data ineffective.

## Acceptance Gate

Do not select v2 based only on internal replacement test MAE. The replacement
distribution is intentionally harder.

Minimum useful signal:

- no meaningful regression on OOD-1000;
- measurable improvement on P8 targeted hard slice;
- Gap MAE must not worsen, because Gap is the most trusted output.
